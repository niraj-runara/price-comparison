# Qwen2.5-72B-Instruct FP8 — unified RTX vs unified L4 (GCP)

Compare two **unified** (non-disaggregated) deployments of the same checkpoint:

| Role | Machine | GPUs | TP |
| --- | --- | --- | --- |
| Unified baseline | `g4-standard-48` | 1× RTX PRO 6000 | 1 |
| Unified L4 | `g2-standard-48` | 4× L4 | 4 |

```
  ┌─────────────────────────┐     ┌─────────────────────────┐
  │ unified-rtx-node         │     │ unified-l4-node          │
  │ g4-standard-48           │     │ g2-standard-48           │
  │ 1x RTX PRO 6000 (tp=1)   │     │ 4x L4 (tp=4)             │
  │ nginx :80 + SGLang :30000│     │ nginx :80 + SGLang :30000│
  └─────────────────────────┘     └─────────────────────────┘
```

Model weights: `gs://gcp-models-bucket/qwen2.5-72b-instruct-fp8/`, synced to
`/mnt/models/qwen2.5-72b-instruct-fp8` on boot. Both servers launch with
`--quantization fp8`. The L4 node also uses `--disable-piecewise-cuda-graph`.

GCP project: `main-entropy-495701-p6`, zone: `us-central1-a` (edit
`cluster/deploy.env` — need a zone with both G4 and G2).

Boot image: `runara-base-sglang` family.

## Layout

```
cluster/deploy.env                              # project, zone, instance names
cluster/deploy.sh                               # push config + start both nodes
cluster/startup-scripts/unified-rtx-node-startup.sh
cluster/startup-scripts/unified-l4-node-startup.sh
config/cluster.env                              # generated by deploy.sh
benchmark/run_benchmark.py                      # sweeps concurrency x {rtx, l4}
benchmark/cost_model.py
benchmark/plot_results.py
```

## Quick start

```bash
# 1. Create VMs
gcloud compute instances create unified-rtx-node \
  --project=main-entropy-495701-p6 --zone=us-central1-a \
  --machine-type=g4-standard-48 --maintenance-policy=TERMINATE \
  --image-family=runara-base-sglang --image-project=main-entropy-495701-p6

gcloud compute instances create unified-l4-node \
  --project=main-entropy-495701-p6 --zone=us-central1-a \
  --machine-type=g2-standard-48 --maintenance-policy=TERMINATE \
  --image-family=runara-base-sglang --image-project=main-entropy-495701-p6

# 2. Deploy (syncs model, installs systemd + nginx, waits for /health)
chmod +x cluster/deploy.sh
./cluster/deploy.sh
```

`g4-standard-48` = 1× RTX PRO 6000; `g2-standard-48` = 4× L4 — no separate
`--accelerator` flag.

## Benchmark

```bash
gcloud compute instances describe unified-rtx-node --zone=us-central1-a \
  --project=main-entropy-495701-p6 \
  --format='get(networkInterfaces[0].accessConfigs[0].natIP)'
gcloud compute instances describe unified-l4-node --zone=us-central1-a \
  --project=main-entropy-495701-p6 \
  --format='get(networkInterfaces[0].accessConfigs[0].natIP)'
```

```bash
pip install -r benchmark/requirements.txt

# Copy benchmark/benchmark.env.example -> benchmark/benchmark.env and set IPs
python3 benchmark/run_benchmark.py
python3 benchmark/plot_results.py
```

Sweeps concurrency `1, 5, 10, 20, 40, 80` against both endpoints and writes:

- `benchmark/results/combined_results.json`
- `benchmark/results/plots/throughput.png`
- `benchmark/results/plots/latency_percentiles.png`
- `benchmark/results/plots/cost_per_1m_tokens.png`

Hourly costs in `benchmark/cost_model.py` are approximate on-demand list prices
(~$4.50/hr for `g4-standard-48`, ~$4.00/hr for `g2-standard-48`) — verify before
trusting cost/1M numbers.
