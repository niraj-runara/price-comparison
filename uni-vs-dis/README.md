# Qwen2.5-72B-Instruct FP8 — unified vs disaggregated (GCP)

Compare a single-GPU **unified** baseline against a **prefill/decode
disaggregated** cluster on the same model checkpoint.

## Architecture

```
                       ┌─────────────────────────────┐
                       │  cpu-node (n2-standard-8)    │
                       │  nginx :80                   │
                       │  sglang_router :8000         │
                       └───────────┬─────────┬────────┘
                                   │         │
                     prefill:30000 │         │ decode:30001
                                   ▼         ▼
                 ┌───────────────────┐   ┌───────────────────┐
                 │ prefill-node       │   │ decode-node        │
                 │ g2-standard-48     │   │ g2-standard-48     │
                 │ 4x L4 (tp=4)       │   │ 4x L4 (tp=4)       │
                 └───────────────────┘   └───────────────────┘

  unified-node (g4-standard-48, 1x RTX PRO 6000, tp=1) — standalone baseline,
  runs prefill+decode combined on one GPU, not part of the disaggregated set.
```

| Role | Machine | GPUs | TP |
| --- | --- | --- | --- |
| Unified baseline | `g4-standard-48` | 1× RTX PRO 6000 | 1 |
| Prefill | `g2-standard-48` | 4× L4 | 4 |
| Decode | `g2-standard-48` | 4× L4 | 4 |
| Router / nginx | `n2-standard-8` | — | — |

Model weights: `gs://gcp-models-bucket/qwen2.5-72b-instruct-fp8/`, synced to
`/mnt/models/qwen2.5-72b-instruct-fp8` on every GPU node at boot. Workers launch
with `--quantization fp8`. L4 PD workers also use `--disable-piecewise-cuda-graph`.

GCP project: `main-entropy-495701-p6`, zone: `us-central1-a` (set in
`cluster/deploy.env` — pick a zone where both G4 and G2 are available).

Boot image: `runara-base-sglang` family (SGLang preinstalled).

## Boot order

1. prefill worker (prefill-node)
2. decode worker (decode-node)
3. `router` + `nginx` (cpu-node)

Enforced two ways:
- **Self-healing**: every service's systemd unit retries forever
  (`StartLimitIntervalSec=0`) on failure so any instance can reboot and the
  cluster re-converges.
- **Explicit control**: [`cluster/orchestrate.sh`](cluster/orchestrate.sh)
  SSHes into each instance in the order above, restarts the relevant service,
  and blocks on its health check before moving to the next.

## Layout

```
cluster/deploy.env                        # <- you edit this: project, zone, instance names
cluster/deploy.sh                         # <- you run this: pushes config, sets up, brings cluster up
cluster/deploy-unified.sh                 # optional: set up only the RTX unified baseline
config/cluster.env                        # generated by deploy.sh — don't hand-edit
cluster/startup-scripts/cpu-node-startup.sh
cluster/startup-scripts/prefill-node-startup.sh
cluster/startup-scripts/decode-node-startup.sh
cluster/startup-scripts/unified-node-startup.sh
cluster/orchestrate.sh                    # up | down | status | restart
benchmark/load_test.py
benchmark/cost_model.py
benchmark/run_benchmark.py
benchmark/plot_results.py
```

## *** Verify before running against real infra ***

1. **SGLang flags.** Confirm `--disaggregation-mode`,
   `--disaggregation-transfer-backend`, and router PD flags match the image
   (`python3 -m sglang.launch_server --help` / `sglang_router.launch_router --help`).
2. **Health endpoints.** Workers/router are assumed to expose `GET /health`.
3. **Baseline hourly** in `benchmark/cost_model.py` (`BASELINE_HOURLY_USD`) is
   approximate on-demand for the RTX node — override with
   `plot_results.py --baseline-hourly` when you have a better number.
4. **Internal DNS**: instances must share one VPC so short hostnames resolve.

## Quick start

Prereqs: `gcloud` CLI authenticated to `main-entropy-495701-p6`, IAP tunneling
enabled (deploy uses `--tunnel-through-iap`).

```bash
# 1. Create instances (skip any that already exist)
gcloud compute instances create cpu-node \
  --project=main-entropy-495701-p6 --zone=us-central1-a \
  --machine-type=n2-standard-8 --image-family=runara-base-sglang \
  --image-project=main-entropy-495701-p6

gcloud compute instances create prefill-node decode-node \
  --project=main-entropy-495701-p6 --zone=us-central1-a \
  --machine-type=g2-standard-48 --maintenance-policy=TERMINATE \
  --image-family=runara-base-sglang --image-project=main-entropy-495701-p6

# unified baseline: 1x RTX PRO 6000
gcloud compute instances create unified-node \
  --project=main-entropy-495701-p6 --zone=us-central1-a \
  --machine-type=g4-standard-48 --maintenance-policy=TERMINATE \
  --image-family=runara-base-sglang --image-project=main-entropy-495701-p6

# 2. Edit cluster/deploy.env if names/zone differ (defaults match above)

# 3. Deploy + bring up
chmod +x cluster/deploy.sh cluster/deploy-unified.sh cluster/orchestrate.sh
./cluster/deploy.sh

# 4. Check
./cluster/orchestrate.sh status
```

`g2-standard-48` bundles **4× L4**; `g4-standard-48` bundles **1× RTX PRO 6000** —
no separate `--accelerator` flag. Prefill/decode use `--tp-size 4`; unified uses
`--tp-size 1`.

First deploy takes a while: each GPU node syncs
`gs://gcp-models-bucket/qwen2.5-72b-instruct-fp8/` before workers start. When
status shows `active`, hit the cluster via cpu-node external IP on port 80 (or
router `:8000`):

```bash
gcloud compute instances describe cpu-node --zone=us-central1-a \
  --project=main-entropy-495701-p6 \
  --format='get(networkInterfaces[0].accessConfigs[0].natIP)'
```

To set up **only** the unified baseline (without touching PD):

```bash
./cluster/deploy-unified.sh
```

## Deploy notes

Re-run `./cluster/deploy.sh` any time after editing `deploy.env` or a startup
script — it's idempotent. For state/restart without redeploying config:

```bash
./cluster/orchestrate.sh status
./cluster/orchestrate.sh restart
```

If instances aren't in the same VPC, replace hostnames in the generated
`config/cluster.env` with internal IPs (or bake overrides into `deploy.sh`'s
heredoc).

## Benchmark

```bash
gcloud compute instances describe cpu-node --zone=us-central1-a \
  --project=main-entropy-495701-p6 \
  --format='get(networkInterfaces[0].accessConfigs[0].natIP)'
gcloud compute instances describe unified-node --zone=us-central1-a \
  --project=main-entropy-495701-p6 \
  --format='get(networkInterfaces[0].accessConfigs[0].natIP)'
```

```bash
pip install -r benchmark/requirements.txt

# Copy benchmark/benchmark.env.example -> benchmark/benchmark.env and set IPs
python3 benchmark/run_benchmark.py
python3 benchmark/plot_results.py
```

Sweeps concurrency `1, 5, 10, 20, 40, 80` against both endpoints and writes
throughput/latency JSON. Cost analysis is **post-hoc** in `plot_results.py`:
only the RTX baseline hourly price is hardcoded (`BASELINE_HOURLY_USD` or
`--baseline-hourly`). For each concurrency it solves for the disaggregated
cluster’s break-even `$/hr` that would match the baseline’s `$/1M` tokens:

```text
break_even_disagg_$/hr = baseline_$/hr * (tps_disagg / tps_unified)
```

Re-run with a different baseline price without re-benchmarking:

```bash
python3 benchmark/plot_results.py --baseline-hourly 5.00
```

Outputs:

- `benchmark/results/combined_results.json`
- `benchmark/results/breakeven.json`
- `benchmark/results/plots/throughput.png`
- `benchmark/results/plots/latency_percentiles.png`
- `benchmark/results/plots/break_even_hourly.png`
